# Security Operations Centre (SOC) Responsibilities

This section outlines the core responsibilities of a Security Operations Centre (SOC) analyst and explains how Splunk supports day‑to‑day monitoring, investigation, and incident response activities.

## 1. Key SOC Responsibilities

### Monitoring Network Traffic
SOC analysts continuously monitor logs, network flows, authentication events, and system activity to detect suspicious behaviour early.

### Investigating Alerts
Alerts generated by SIEM tools must be validated, triaged, and escalated when necessary.  
Analysts determine whether an alert is a false positive or a genuine threat.

### Responding to Incidents
When a threat is confirmed, SOC analysts take action by:
- Containing affected systems  
- Blocking malicious IPs  
- Resetting compromised accounts  
- Coordinating with IT teams  

### Analysing Logs and Events
Analysts review logs from multiple sources to identify patterns, anomalies, and indicators of compromise (IOCs).

### Maintaining Security Tools
SOC teams maintain and tune tools such as:
- SIEM platforms  
- IDS/IPS  
- Endpoint protection  
- Firewalls  
- Threat intelligence feeds  

This ensures accurate alerting and reliable visibility across the environment.

## 2. How Splunk Supports SOC Operations

Splunk plays a central role in SOC workflows by collecting, normalising, and analysing data from multiple sources, including:

- **Firewalls**  
- **IDS/IPS systems**  
- **Access control systems**  
- **Authentication logs**  
- **Servers and endpoints**  
- **Cloud platforms**  

By aggregating this data into a single platform, Splunk enables analysts to:

- Detect threats quickly  
- Investigate incidents efficiently  
- Correlate events across systems  
- Build dashboards and reports  
- Automate repetitive tasks  

Splunk’s search and visualisation capabilities make it one of the most widely used SIEM tools in modern SOC environments.

## 3. Why SOC Work Matters

SOC analysts are the front line of defence for an organisation.  
Their work helps prevent:

- Data breaches  
- Account compromise  
- Malware outbreaks  
- Insider threats  
- Service disruptions  

This section demonstrates an understanding of real‑world SOC duties and how tools like Splunk support effective security monitoring.
